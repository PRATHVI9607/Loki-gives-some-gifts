# AI Tools + Claude Ecosystem Hub

A clean, organized collection of high-value links for Claude skills, plugins, MCP tooling, open-source agents, and setup workflows.

This hub is designed as a practical "build your stack" library: pick what you need, skip what you don’t, and evolve your own stable toolkit.

## What’s in this repo

- `links/01-official-claude-and-anthropic.md` → Official Anthropic/Claude resources
- `links/02-skills-and-plugin-collections.md` → Skills repos, awesome lists, and plugin catalogs
- `links/03-open-source-agents-and-tooling.md` → Open-source agent frameworks, MCP tools, and dev utilities
- `links/04-creative-marketing-and-growth.md` → Design, marketing, SEO, and growth-related links
- `links/05-setup-guides-and-workflows.md` → Practical setup instructions and workflows
- `links/06-evals-observability-and-safety.md` → Evaluation, tracing, and safety/security references
- `links/07-learning-communities-and-updates.md` → Learning sources, changelogs, and communities
- `links/08-claude-code-complete-guide.md` → Claude Code commands, features, token optimization, and key ecosystem links
- `CLAUDE_AGENT_DIRECTIVES.md` → Cleanly formatted directives and execution rules

## Quick start (best setup path)

1. Start with official resources in `links/01-official-claude-and-anthropic.md`.
2. Install/adapt skills using `https://claude.ai/customize/skills`.
3. Pick 1–2 skill collections from `links/02-skills-and-plugin-collections.md`.
4. Add one MCP/tooling repo from `links/03-open-source-agents-and-tooling.md`.
5. Use the setup checklist in `links/05-setup-guides-and-workflows.md`.
6. Add eval + tracing from `links/06-evals-observability-and-safety.md`.
7. Use `links/08-claude-code-complete-guide.md` as your day-to-day operating guide.
8. Use `links/05-setup-guides-and-workflows.md` Workflow G/H for multi-tool and legal/public pack setup.

## Notes

- Links were deduplicated and grouped by purpose.
- Tracking query params (`utm_*`) were removed where possible.
- A few entries are short links (`lnkd.in`) from the original source and are preserved as-is.
- Prefer starting with 1 tool per category, then expanding only after each addition proves useful.
